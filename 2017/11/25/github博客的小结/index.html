github博客的小结

发表于 2017-11-25

这段时间零零散散的花了些时间来弄一个博客，由hexo+github pages搭建而成，算下来竟然也花了不少时间，哎哎，不知道自己怎么就掉坑了去了。这篇博客主要是自己的总结了，总结自己的搭建过程以及以后需要的一些操作。如果想知道怎么搭建博客的童鞋我推荐去看看另一个教程吧，这篇文章就写了些简短的思路。

手把手教你使用Hexo + Github Pages搭建个人独立博客

首先安装node环境，然后就是安装hexo。接下来就是按照上面的教程开始配置，找了一个最常用的主题，然后修改一些参数，再去github上新建一个仓库。clone下来。本来再改改就能用了。然而我没有往下弄了。然后忙着别的事就把这事给忘了（记忆力感人）。下次再记起的时候又重新找了一个教程。然后继续往下干，然后在又clone了一遍,之前clone在.deploy,这个一般情况下看不到。接着悲剧就出来了。我之后一直都是在source里面写完，然后自己拷贝到git仓库里面。再push上去，之后就是发现在博客的地址上始终找不到自己新写的博客（哇哇）。然后各种查教程，最后竟然又找到了上面那篇，才发现自己已经写了这个自动的脚本。然后再改了一个在_config.yml里面改了auto_excerpt就可以展示了。

接下来记一下自己写博客的操作吧，真怕自己好久不写又忘记了。

  1. hexo new <文件名> (不需要.md)
  2. 然后在sublime里面打开开始写吧。
  3. 执行自动化脚本　source deploy.sh

然后就可以愉快的看博客了。

附：
几个命令
hexo -d :部署
hexo -g :生成
